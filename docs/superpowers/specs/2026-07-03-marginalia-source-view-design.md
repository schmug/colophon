# Marginalia Source View — design

**Date:** 2026-07-03
**Status:** approved (design), pending implementation plan
**Repo:** Colophon (`marginalia.py`, `test_marginalia.py`)

## Problem

Marginalia's source-echo panel proves a prompt appears verbatim in the training
corpus and reports where — `find_source_echo()` returns `file` + `line`, and
`renderSource()` ([marginalia.py:497](../../../marginalia.py)) prints them as
`018_argon.yaml:2`. But that label is dead text. The project's core claim is
"every property you normally estimate is here ground truth you can audit,
because the corpus is readable" — yet from the UI, the corpus is *not*
readable. The audit loop stops one click short of the artifact.

## Goal

Make the `file:line` label a real link that opens **the exact training file
the model saw**, served by Marginalia itself from the corpus already loaded in
memory — new tab, matched line highlighted and scrolled into view. Ground
truth, offline, stdlib-only.

Non-goals (explicitly rejected in brainstorming): external links to
Codeberg/GitHub (the local clone is the snapshot the model trained on; the
live index diverges), and a browsable corpus index (YAGNI — the only entry
point is the source-echo match).

## Design

### Route: `GET /source?mode=<id>&file=<name>&line=<n>`

A server-rendered HTML page, deliberately **not** under `/api/` (it returns a
page, not JSON). Handled in `make_handler()`'s `do_GET` alongside the existing
routes.

- **Lookup is in-memory only.** The filename is matched by exact string
  equality against the mode's `(name, text)` list that `_load_mode()` built at
  startup via `load_corpus_files()`. No `open()`, no path join, no filesystem
  access at request time — path traversal is impossible by construction, not
  by sanitization.
- **Error semantics mirror the existing routes:**
  - unknown `mode` → **400** (like `/api/analyze`)
  - known mode whose weights/corpus are absent → **503** (like `/api/analyze`)
  - `file` not in that mode's corpus → **404**
  - non-integer `line` → **400** (matching `/api/saliency`'s strict `pos`)
  - out-of-range `line` → **200**, rendered with no highlight (cosmetic param,
    not worth an error)
  - missing `line` → **200**, no highlight
- Error responses are small HTML pages with the correct status code (this is a
  browser tab, not a fetch target).

### Rendered page

- Same monochrome/serif look as the main page (shared inline CSS, kept
  minimal — this page needs no JavaScript at all).
- `<title>` and header: mode label + filename (e.g. "Periodic table —
  018_argon.yaml").
- Body: the file's exact text, **HTML-escaped** (`html.escape` — corpus bytes
  go into markup; non-negotiable), one row per line with line numbers, each
  row anchored `id="L<n>"`.
- When `line` is valid, that row gets a highlight class; the link from the
  main page carries a `#L<n>` fragment so the browser scrolls natively.

### Provenance footer

Each mode gains a `source_note` entry in `MODE_META`, rendered at the bottom
of the source view:

- **osai** — European Open Source AI Index, CC-BY 4.0,
  doi:10.5281/zenodo.15386042, link to the Codeberg repo. Serving these files
  verbatim makes attribution mandatory (repo's own attribution reminder), and
  the note observes that the local clone is a snapshot.
- **elements** — undisputed IUPAC facts, generated by the committed
  `teaching_data/build_elements.py`.
- **kana** — traditional Hepburn chart, generated by the committed
  `teaching_data/build_kana.py`.

The footer also shows the corpus sha256 snapshot (already computed at load by
`corpus_sha256()`) — a colophon on the colophon's corpus.

### Frontend change (one function)

`renderSource()` wraps the existing `file:line` text in
`<a href="/source?mode=<activeMode>&file=<file>&line=<line>#L<line>"
target="_blank" rel="noopener">` (URL-encoded params, DOM-built — no innerHTML
with data in it). No-match behavior unchanged. No other UI changes.

### Plumbing

- `make_handler()`'s per-mode config already carries `files`; the source page
  builder needs the mode label and `source_note` too — both come from
  `MODE_META`, which the handler already reads.
- A module-level `source_page(cfg, filename, line)` (or similar) returns the
  HTML string, so tests can hit it via the live server *and* the route stays a
  thin dispatcher, matching the existing style.

## Testing (`test_marginalia.py`, live ephemeral-server pattern)

1. Valid request → 200; body contains the file's text escaped, line numbers,
   and the highlight markup on the requested line.
2. Unknown mode → 400; absent mode → 503; unknown file → 404; non-integer
   `line` → 400.
3. Out-of-range `line` → 200 with no highlight markup.
4. Escaping: a fake in-memory corpus entry containing `<script>` renders
   escaped (no raw `<script>` in the body).
5. Provenance: the osai source view contains the CC-BY/doi attribution.
6. The served main page's JS contains the `/source?` link-building code, and
   `renderSource`'s output is an anchor (asserted on the page source, since JS
   isn't executed under unittest).

## Constraints

- Stdlib-only (`http.server`, `html`, `urllib.parse`) — no new dependencies,
  per Marginalia's design intent.
- One missing corpus never takes the page down (same degradation contract as
  every other route).
- TDD: failing tests first, then the route (global workflow rule).
