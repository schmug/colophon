# Plan: Periodic-table teaching corpus + Marginalia teaching mode

Slug: `elements-teaching-corpus`
Status: approved (recommendations accepted 2026-07-01)

## Problem

A layperson can't tell whether Colophon's confidence / off-map signals are
trustworthy, because the only corpus (the OSAI index) is jargon they can't
grade by eye — they take "I'm sure" on faith, the exact thing Colophon exists
to abolish. A second corpus whose ground truth already lives in the reader's
head (the periodic table) lets them validate the machinery against their own
knowledge before trusting it on the corpus they *can't* check.

## Decisions (locked)

1. **Corpus layout:** 118 per-element YAML files under `teaching_data/elements/`,
   generated by a committed script from one embedded, IUPAC-cited table; the
   generated files are checked in so `--src` works offline. Mirrors
   `sample_data/` and keeps `find_source_echo` per-entry clean.
2. **Second-model mechanism:** add an `--out` (weights path) option to
   `colophon.py` so a second model saves to `elements.npz` without clobbering
   `colophon.npz`; the colophon-json path is derived from it. Default behavior
   unchanged.
3. **Element schema:** minimal + small ints — `number:`, `symbol:`, `name:`,
   `period:`, `group:`. No atomic masses (would inflate the digit vocab).
4. **Marginalia default mode:** OSAI (flagship, preserves the theme); elements
   is the opt-in "teaching" toggle.

## Scope

**In:** elements corpus + committed generator; `--out` plumbing on colophon.py;
Marginalia teaching-mode toggle (two models/corpora served, `/api/analyze?mode=`
routes, absent mode → 503); tests; README + CLAUDE.md updates; IUPAC attribution.

**Out:** no change to the MLP / transformer / confidence math; no change to the
OSAI scorecard (grades the artifact, not the corpus); no third corpus;
`colophon.py demo` stays OSAI-only.

## Acceptance criteria

1. `python colophon.py --src teaching_data/elements prepare` → 118 entries,
   vocab ⊆ `[A-Za-z0-9 :.\n-]`, data section has num_files/num_characters/sha256.
2. Trained elements model (on-theme, revised after measuring): (a) a foreign
   character sets the off-map flag; (b) an all-in-vocab fake prompt
   (`number: 250\n`) does NOT set the off-map flag AND its entropy is not
   meaningfully higher than a real element's — demonstrating the load-bearing
   lesson that confidence under-reads OOD and only the categorical flag is
   trustworthy; (c) a low-temp continuation from `number: 26\n` reproduces the
   self-evident correct facts "Fe" and "Iron". (test)
   NOTE: the original "entropy(26) < entropy(250)" criterion was dropped — it
   contradicted the project thesis and only passed by luck (measured: 0.056 vs
   0.065, but `number: 999` was 0.051, lower than a real element).
3. Elements model saves to `elements.npz`, `colophon.npz` untouched. (test)
4. Marginalia `/api/analyze?mode=elements` routes to elements model+corpus;
   `mode=osai`/default routes to OSAI; absent mode → 503 clear message. (test)
5. New tests + existing `test_colophon.py` / `test_marginalia.py` pass.
